# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Heisenberg, please report it responsibly.

**Email:** kontakt@kamilpajak.pl

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact

## Response Timeline

- **Acknowledgment:** within 48 hours
- **Initial assessment:** within 1 week
- **Fix or mitigation:** best effort, typically within 30 days

## Scope

This policy covers the open-source CLI (`cmd/cli`, `pkg/*`) and the GitHub Action. SaaS components (`internal/auth`, `internal/billing`, `internal/database`) are covered separately.

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest release | Yes |
| Previous minor | Best effort |
| Pre-release | No |
